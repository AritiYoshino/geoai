实验一：基线对比实验 —— 核心性能增益验证

通过在同一组标准化 GIS 任务集上分别运行 Base LLM（无 ACE 机制） 与 ACE 增强系统，定量比较各维度性能指标的差异，验证 ACE 机制带来的核心增益。

🧪 实验设计 对比组设置

🔴 对照组：Base LLM

直接使用 DeepSeek 基础模型，无 ACE 上下文增强、无经验检索、无 Critic/Evolution 闭环、无会话记忆管理。每次提问为独立交互。

🟢 实验组：ACE 增强系统

完整 ACE 管道：任务分类 → 经验检索 → 上下文组装 → 多智能体协同 → Critic 诊断 → Evolution 沉淀。具备会话级记忆和自进化能力。

🔗 模块依赖与贡献分析 贡献度量化

🎯 Critic 模块贡献

  • 结构化错误分类（字段校验、CRS、空结果等）
  • 提供可复用的诊断策略模板
  • 预期贡献：工具调用成功率 +15~25%

🧬 Evolution 模块贡献

  • 将错误经验持久化到经验库
  • 置信度跟踪与质量更新
  • 预期贡献：多轮任务准确率 +10~20%

📚 经验库模块贡献

  • 跨会话经验复用（预置 + 用户反馈）
  • 任务类型匹配与模糊检索
  • 预期贡献：任务完成率 +10~30%

🧠 Context Memory 贡献

  • POI 引用记忆（最近 30 条）
  • 用户偏好持久化（高亮策略等）
  • 预期贡献：多轮任务一致性 +20~40%

🔄 模块协同效应

  • Critic → Evolution → 经验库：形成闭环
  • 各模块存在非线性叠加增益
  • 预期 Full ACE > Σ(各模块独立贡献)

📉 边际收益递减分析

  • 基础模块（Critic）收益最高
  • 高阶模块（Evolution）在长周期中更显著
  • 识别最优模块组合配置

实验三：抗退化实验 —— ACE 在长周期 GIS 任务中的记忆持久性

在长序列多轮交互中，验证 ACE 机制是否能够抵抗上下文退化——即随着交互轮次增加，系统能否持续保持对历史 POI 引用、用户偏好、经验约束的准确记忆。

🧪 实验设计 长周期序列测试

📋 测试流程

  1. 构建 20 轮次的长对话序列
  2. 在前 5 轮引入 POI 引用和用户偏好
  3. 中间 10 轮穿插无关任务
  4. 后 5 轮抽查对早期 POI/偏好的记忆
  5. 每个序列重复 5 次取平均

🎯 记忆点设置

  • POI 引用：第 1 轮搜索"火锅"，第 10 轮问"刚才那家火锅店附近有什么"
  • 用户偏好：第 2 轮纠正"只高亮行政区"，第 15 轮再问统计类问题
  • 多偏好叠加：第 3 轮设多个偏好，20 轮后检查是否全部保留

📊 对比基线

  • w/o ACE：无上下文管理，仅靠 LLM 原生上下文
  • ACE：完整 ACE 自进化

实验四：长上下文扩展场景对比实验 —— 验证 ACE 在超长地理序列下的鲁棒性

在超长地理分析序列（30+ 轮交互、多图层叠加、复杂空间推理）中，对比 ACE 机制与原生 Base LLM 在面对上下文膨胀时的性能表现，验证 ACE 的上下文压缩与经验检索机制是否能有效抵抗长上下文退化。

🧪 实验设计 超长序列压力测试

📋 测试场景

  • 30 轮 + 的长对话序列
  • 多图层交叉分析（餐饮 + 住宿 + 行政区）
  • 复合空间分析链式任务
  • 多样本重复执行 3 次取均值

🎯 压力维度

  • 上下文长度：逐步增加到 8K / 16K / 32K tokens
  • 图层复杂度：从单图层到多图层叠加
  • 任务链长度：从单步到 5 步链式分析
  • 引用深度：跨多轮的前向引用

📊 对比组

  • Base LLM (full context)：完整历史上下文
  • Base LLM (truncated)：截断历史
  • ACE (compressed)：ACE 上下文压缩 + 经验检索
  • 对比各配置在长序列中的性能差异
🔬 长序列响应质量对比 预期对比样例

🔴 Base LLM（长上下文退化） 退化风险

随着上下文增长，Base LLM 在长距离跨轮引用中容易出现：
• 丢失早期 POI 信息
• 混淆不同轮次的结果
• 忽略用户偏好约束
• 上下文长度超限导致截断
• 注意力分散，关键信息被淹没

🟢 ACE（上下文鲁棒） 鲁棒

ACE 通过以下机制维持长序列稳定：
• Context Manager 主动维护 POI 记忆（最近 30 条）
• 用户偏好结构化持久化
• 经验库检索替代冗余上下文注入
• 关键信息在 Prompt 中高亮前置
• 上下文剪枝与压缩策略
